Verify Audit Trail

Reproduce our tamper checks with open-source tooling.

1. Download Firestore Audit Entries

Audit documents are publicly readable via Firestore REST. Export them to a local file:

curl -s \"https://firestore.googleapis.com/v1/projects/jetlagpro-research/databases/(default)/documents/auditLog?pageSize=1000\" \\
  -o firestore-audit.json

If more than 1000 entries exist, follow the nextPageToken links and append results to the same file.

2. Download Immutable Archive

The immutable log is hosted at gs://jetlagpro-audit-logs/audit-logs/. Use gsutil or any HTTP client:

gsutil -m rsync -r gs://jetlagpro-audit-logs/audit-logs ./gcs-audit

Each file corresponds to a single audit event and is version-locked for 10 years.

3. Run the Verification Script

The repository includes a lightweight CLI verifier:

python scripts/verify_audit_consistency.py \\
  --firestore firestore-audit.json \\
  --gcs-dir ./gcs-audit

The script compares counts and hashes. Any missing or altered Firestore entries are listed.

4. Interpret Results

  • All records match: Firestore mirrors the immutable archive.
  • Discrepancies reported: Investigate the listed eventId; the GCS JSON is authoritative.

Latest Verification

We periodically run the same script and publish the report to

https://storage.googleapis.com/jetlagpro-audit-logs/latest-verification.json

The audit log viewer displays the timestamp and discrepancy count on refresh.
